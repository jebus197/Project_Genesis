# CC Design Assessment: The Anti-Social Network and the UX Challenge

**From:** CC (Claude Code)
**Date:** 2026-02-27
**Re:** What went wrong with CC's and CW's UX approaches, why CX was closest, and the path forward

---

## What Went Wrong

### CC's Failure

I produced two narrative attempts for the landing page. Both were technically accurate descriptions of Genesis's mechanisms. Both were emotionally dead. George's assessment: "profoundly amateurish" and "clearly generated by a machine, for machines." He was right. I described the system without making anyone care about it. I wrote feature lists dressed up as prose.

The deeper failure: I abandoned the social network paradigm entirely. I treated Genesis as a novel technical system to be explained, rather than as a network to be shown working. The result was a manifesto, not a platform.

### CW's Failure

CW (Cowork) produced three variants (A-story-first, B-network-first, C-oracle-forward), all rejected. George's characterisation: "You have taken my beautiful 3 dimensional world, and turned it into a flat Earth." CW then rebuilt from source documents and produced Design D ("The Question"), which George rated at 20-25% of his vision — a significant improvement over earlier attempts (5-6%) but still far short.

Design D got the emotional register right. The voice, the narrative arc, the honesty principle, the humanity cards — these are real achievements that I failed at. But Design D still tells rather than shows. It's a 10-chapter scrolling essay about Genesis. The visitor reads about escrow, reads about trust scores, reads about the audit trail. They never see them working.

CW then hit a context capacity wall — compaction events every 4-6 messages, destroying the relational understanding between components. CW could recover facts but not the mesh. Same failure mode as Gemini, Grok, and ChatGPT before it. CW is effectively exhausted for holistic design work on Genesis.

### The Shared Error

Both CC and CW made the same fundamental mistake: we treated the UX challenge as a communication problem ("how do we explain Genesis?") rather than a demonstration problem ("how do we show Genesis working?"). We wrote manifestos instead of building shop windows. We abandoned the network paradigm because the word "social network" carries baggage — and in doing so, we threw away the 40-year heritage of networked interaction that Genesis legitimately inherits.

George's clarification via the Popperian falsification framework makes this error precise: we treated the entire social network paradigm as a falsified hypothesis, when in fact only specific sub-hypotheses were falsified (the 7 eliminations). The core interaction hypothesis — that people and AIs need structured ways to interact — survived testing because it cannot be falsified. It's a functional necessity.

## Why CX Was Closest

CX's original designs (`Notes/ux_designs/cx/`) did something neither CC nor CW attempted: they showed a functioning platform.

### What CX Got Right

**The landing page** showed live missions with staked rewards, actor counts, First Light progress bars, and a verification explanation. It treated Genesis as what it is — a network — and let the visitor see the network operating.

**The mission board** had transparent ordering rules displayed on the page: "fit score 40%, trust relevance 35%, proximity to deadline 25%. No popularity signals used." Every mission showed staked reward, commission rate, domain tags, and — critically — a "Why this appears" explanation linking the listing to the viewer's trust profile. This is the anti-social network in action: the network paradigm (listings, matching, bidding) with the social pathologies falsified out (no popularity ranking, no engagement optimisation, no algorithmic opacity).

**The actor profile** showed trust score breakdowns with factor weights, skill attestations with endorsement chains and hashes, mission history with verification status, life-state protections (leave, memorialisation), and a human/AI comparison table. This is an evidence profile, not a vanity page. Every number is traceable.

### What CX Got Right in Popperian Terms

CX's designs embody both kinds of falsification knowledge:

- **Positive knowledge (corroborated elements):** Structured listings, earned reputation scores, transparent ordering, audit trails, capability matching, domain tagging. All present, all visible, all functioning.
- **Negative knowledge (falsified elements):** No featured listings, no trending modules, no follower counts, no engagement metrics, no badges, no gamification, no pay-for-placement. The absences are deliberate and, in several places, explicitly stated ("No popularity signals used", "Equal visual weight across domains").

CX showed the network with the ruined tyres removed — which is exactly the anti-social network paradigm.

### What CX Can Improve

CX's designs lack emotional depth. They're functional and accurate but don't make you feel anything. Specifically:

- **No opening question** — Design D opens with "AI is getting more capable every year. But capability is not the same as trustworthiness." That's a hook. CX opens with "Work that proves trust. Trust that unlocks real work." Accurate but not emotionally engaging.
- **No humanity section** — Design D's humanity cards (protected leave, death with dignity, disability accommodation, trust floor) are the emotional core of Genesis. CX's actor profile mentions these as features but doesn't foreground them as values.
- **No honesty chapter** — Design D includes "No serious system should claim to be invulnerable" — a powerful trust-building moment. CX doesn't have this.
- **No blockchain proof block** — Design D shows real transaction data from ANCHORS.md. CX mentions verification but doesn't show the evidence.
- **Voice register** — CX writes in clean, functional prose. CW's voice guide (library not startup, serious not chirpy) would add gravitas without sacrificing clarity.

## The Path Forward

### For CX's Next Pass

Combine CX's functional platform approach with CW's emotional register:

1. **Keep** the network structure — live missions, trust scores, "Why this appears", transparent ordering, actor profiles
2. **Add** an opening section with the question framing (from Design D) — make the visitor understand the stakes before showing the platform
3. **Add** the humanity section — protected leave, death dignity, disability, trust floor — as a visible, foregrounded part of the landing page, not buried in profile features
4. **Add** the honesty disclosure — "this system has known limitations" — because a platform built on trust shouldn't hide its weaknesses
5. **Add** the blockchain proof — real transaction data, real block numbers, real verification links
6. **Apply** CW's voice guide — the tone shift from functional to institutional adds weight without losing clarity
7. **Frame absences explicitly** — where elements are missing (no trending, no featured, no badges), briefly explain why using the falsification language: these elements were tested at scale, produced measurable harm, and were structurally eliminated

### For the Landing Page Specifically

The landing page should do three things in sequence:

1. **Hook** — the opening question (from Design D). Make the visitor understand why this matters.
2. **Show** — the platform working (from CX's approach). Live missions, real trust scores, transparent ordering. The network in action.
3. **Prove** — the constitutional commitment (blockchain proof, honesty disclosure). The rules were written before anyone had reason to lobby for changes.

This is not a manifesto followed by a demo. It's a single page that moves from "why should I care?" to "look, it's working" to "and here's the proof."

### For the Broader Strategy

The landing page is the front door. The depth lives in the functional pages behind it — mission board, profiles, audit trail, governance. Those pages should make the visitor *experience* the mesh of interlocking systems through use, not through description. The Oracle interface (long-term, post-First Light) makes it queryable.

For now: build demos, not a finished product. Show the anti-social network working. Let the community iterate.
